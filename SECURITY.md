# Política de Segurança

Apenas a versão estável mais recente do CodePad recebe correções de segurança. Versões antigas não são mantidas.

Se encontrar uma vulnerabilidade de segurança, não abra uma issue pública, pois isso poderia expor o problema antes da correção. Entre em contacto por e-mail com a mantenedora do projeto (veja o perfil do repositório ou README) ou, se preferir, use as Security Advisories do GitHub: no repositório, clique em Security e depois em Report a vulnerability. Descreva o problema de forma clara: passos para reproduzir, impacto esperado e, se possível, sugestão de mitigação. Dê um prazo razoável (por exemplo, 90 dias) para resposta antes de divulgar publicamente.

Pode esperar confirmação de receção do reporte, avaliação e, quando aplicável, desenvolvimento de correção, comunicação sobre o andamento e a data prevista de release da correção, e crédito ao reportador na release se desejar ser mencionado.

O escopo desta política abrange a aplicação desktop CodePad (Java/JavaFX) e os scripts de empacotamento e instalação incluídos no repositório. Dependências de terceiros devem ser reportadas aos respetivos projetos; podemos ajudar a encaminhar se relevante para o uso neste projeto.

Obrigada por ajudar a manter o CodePad mais seguro.
